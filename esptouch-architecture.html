🔐 累积纠错算法
  • 20 MHz 带宽：0-5% 错误率
  • 40 MHz 带宽：0-17% 错误率
  • 多轮发送提高成功率至 95%+
  • 公式：[1-(1-P)^k]^l
  • P=成功率, k=轮次, l=数据长度
📡 广播 vs 组播
  • 广播：255.255.255.255:7001
  • 组播：234.0.0.1-100:7001
  • 广播：所有设备接收
  • 组播：特定范围设备
  • 默认使用广播模式
⚡ 性能指标
  • Guide Code：4 轮 × 4 包
  • 数据间隔：8ms
  • 单轮时间：约 4.68s
  • 6 轮成功率：99.9998%
  • 总超时：45 秒
⚠️ 已知限制
  • 不支持 5 GHz Wi-Fi
  • 不支持 802.11ac 协议
  • 需要设备在路由器范围内
  • AP 隔离可能导致无响应
  • V1/V2 之间不兼容

CRC8 多项式算法
多项式: 0x8C · 初始值: 0x00
for dividend = 0 to 255:
    remainder = dividend
    for bit = 0 to 7:
        if (remainder & 0x01):
            remainder = (remainder >> 1) XOR 0x8C
        else:
            remainder = remainder >> 1
    crcTable[dividend] = remainder

// 计算时:
value = 0x00
for each byte b:
    data = b XOR value
    value = crcTable[data & 0xFF] XOR (value << 8)
result = value & 0xFF
